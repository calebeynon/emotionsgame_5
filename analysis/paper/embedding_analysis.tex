\begin{abstract}
We construct text embeddings for 4,700 chat messages from a repeated public goods game and use them to identify the linguistic dimensions that distinguish cooperative from non-cooperative states, promise-making from non-promise-making rounds, homogeneous from heterogeneous contribution groups, and liar from non-liar players. Using OpenAI's \texttt{text-embedding-3-small} (1,536 dimensions) and \texttt{text-embedding-3-large} (3,072 dimensions), we compute centroid-based direction vectors, project messages onto these axes, and evaluate predictive power via logistic regression. We find that the promise direction validates strongly (AUC = 0.90), the homogeneity direction captures coordination language, while the cooperative direction captures settled-versus-negotiating communication rather than cooperative intent. We additionally construct round-level and cumulative liar directions from behavior classification labels, testing whether embedding projections can distinguish the chat of players who lie about their contributions. This document records the methodology, critical data decisions, and results.
\end{abstract}

\section{Overview}
%% ============================================================

The experiment is a repeated public goods game with 16 participants per session, grouped into teams of 4. Players have a 25-point endowment and choose how much to contribute to a group fund (multiplier = 0.4). Before each contribution decision, players communicate via free-form text chat. The analysis here asks: \emph{what do embeddings of chat content reveal about the cooperative and promise-making dimensions of player communication?}

We embed chat at two granularities:
\begin{enumerate}
    \item \textbf{Individual message level} (4,700 messages): each chat message embedded separately.
    \item \textbf{Player-round level} (2,298 texts): all messages from a single player within a single round concatenated into one string before embedding.
\end{enumerate}

We then compute direction vectors for five labeling schemes (cooperative state, promise-making, contribution homogeneity, round-level liar, and cumulative liar), project embeddings onto these directions, and evaluate predictive power.

%% ============================================================
\section{Embedding Models and API}
%% ============================================================

We use two OpenAI embedding models:
\begin{itemize}
    \item \texttt{text-embedding-3-small}: 1,536 dimensions. Cost: \$0.002 per 4,700 messages.
    \item \texttt{text-embedding-3-large}: 3,072 dimensions. Cost: \$0.012 per 4,700 messages.
\end{itemize}

Embeddings are computed once and cached as Parquet files. All downstream analysis reads from cache, so the total API cost for the entire pipeline (both models, both granularity levels) was approximately \$0.03.

\section{Data Decisions}
%% ============================================================

\subsection{Chat-Round Pairing Semantics}

In the oTree experiment, chat occurs \emph{after} the contribution decision in each round. In the analysis module, however, chat messages are paired with the contribution they \emph{influenced}---the next round's decision. This means:
\begin{itemize}
    \item Round 1 of each supergame has no associated chat (no prior discussion).
    \item The last round's chat influenced no subsequent contribution and is stored separately as ``orphan chats.''
    \item Embedding projection scores are \texttt{NaN} for round 1 in the panel dataset.
\end{itemize}

\subsection{Label Sources}

\paragraph{Cooperative state.} Defined at the \textbf{player level} from \texttt{player\_state\_classification.csv}: a player is ``cooperative'' if others' total contribution in their group is $\geq 60$ (i.e., the other three players contributed $\geq 20$ on average). This yields 3,800 cooperative and 900 non-cooperative message-level observations, or 1,853 cooperative and 445 non-cooperative player-round observations.

\paragraph{Promise-making.} Defined at the \textbf{player-round level} from \texttt{promise\_classifications.csv}: an LLM classifier (applied in a prior issue) labels whether a player's chat in a given round contains a promise to contribute. This yields 2,195 promise and 2,505 no-promise messages, or 931 promise and 1,367 no-promise player-rounds.

\paragraph{Contribution homogeneity.} Defined at the \textbf{group-round level} from contribution data: a group-round is ``homogeneous'' if the range of contributions (max $-$ min) is $\leq 1$ ECU, and ``heterogeneous'' otherwise. This threshold captures groups where all members contributed (approximately) the same amount, regardless of the contribution level. The label is applied at the group level and merged onto individual player embeddings.

\paragraph{Round-level liar.} Defined at the \textbf{player-round level} from \texttt{behavior\_classifications.csv}: a player ``lied this round'' (\texttt{lied\_this\_round\_20}) if they made a promise to contribute $\geq 20$ but actually contributed $< 20$. This is a round-specific label that can change from round to round. The dataset contains 94 round-liar observations, reflecting substantial class imbalance.

\paragraph{Cumulative liar.} Defined at the \textbf{player-round level} from \texttt{behavior\_classifications.csv}: a player is a ``cumulative liar'' (\texttt{is\_liar\_20}) if they have \emph{ever} lied (by the round-level definition above) in any prior or current round. Once a player is classified as a liar, the label persists for all subsequent rounds. This yields 123 cumulative-liar observations, somewhat less imbalanced than the round-level label.

\subsection{Embedding Granularity}

Two levels were chosen to balance competing needs:
\begin{itemize}
    \item \textbf{Individual messages} preserve fine-grained variation (a player may send multiple messages per round, some containing promises and others not), enabling message-level ranking by projection score.
    \item \textbf{Player-round concatenation} captures the full conversational context of a player's round-level communication, producing one embedding per player-round that aligns naturally with the player-round-level labels.
\end{itemize}

\subsection{Cross-Level Projections}

For the message-level output, we compute \emph{four} projection scores per message:
\begin{enumerate}
    \item Message projected onto the \textbf{message-level direction vector} (centroid difference computed from individual message embeddings).
    \item Message projected onto the \textbf{player-round direction vector} (centroid difference computed from player-round embeddings).
\end{enumerate}
This is done for both the small and large models, yielding four columns. The rationale is that the player-round direction may be a better axis (since labels are defined at the player-round level), but we want to see both projections at the message level for comparison.

%% ============================================================
\section{Methodology}
%% ============================================================

\subsection{Centroid Direction Vector}

For a given label scheme (cooperative/non-cooperative or promise/no-promise), we compute:
\begin{align}
    \mathbf{c}_+ &= \frac{1}{n_+} \sum_{i \in \text{positive}} \mathbf{e}_i \\
    \mathbf{c}_- &= \frac{1}{n_-} \sum_{i \in \text{negative}} \mathbf{e}_i \\
    \mathbf{d} &= \frac{\mathbf{c}_+ - \mathbf{c}_-}{\|\mathbf{c}_+ - \mathbf{c}_-\|}
\end{align}
where $\mathbf{e}_i$ is the embedding vector for observation $i$, and $\mathbf{d}$ is the unit-length direction vector pointing from the negative class centroid toward the positive class centroid.

\subsection{Projection Score}

Each embedding is projected onto the direction vector:
\begin{equation}
    s_i = \mathbf{e}_i \cdot \mathbf{d}
\end{equation}
Higher values indicate the embedding is more aligned with the positive class direction.

\subsection{Probe Phrase Validation}

To validate that the direction vector captures the intended construct, we embed a set of predefined probe phrases and compute their cosine similarity with $\mathbf{d}$:
\begin{equation}
    \text{sim}(\mathbf{p}, \mathbf{d}) = \frac{\mathbf{p} \cdot \mathbf{d}}{\|\mathbf{p}\|}
\end{equation}
where $\mathbf{p}$ is the probe phrase embedding. If the direction is meaningful, cooperative/promise-related probes should score higher than non-cooperative/non-promise probes.

\subsection{Logistic Regression Baseline}

We run 5-fold stratified cross-validated logistic regression at the \textbf{group-round level} (667 group-rounds), comparing four feature sets:
\begin{enumerate}
    \item \textbf{VADER only}: 4 sentiment features (compound, positive, negative, neutral means).
    \item \textbf{Projection only}: the 1-dimensional projection score from the direction vector.
    \item \textbf{Full embedding (PCA 50)}: the raw embedding reduced to 50 principal components.
    \item \textbf{VADER + Projection}: sentiment features combined with the projection score.
\end{enumerate}

For cooperative state, the group-round label is assigned by majority vote of player-level states. For promise-making, the group-round label is assigned by majority vote of player-level \texttt{made\_promise} booleans.

\section{Results: Cooperative Direction}
\label{sec:coop}
%% ============================================================

\subsection{Direction Vector Problem}

The cooperative direction vector does \emph{not} capture cooperative intent. Instead, it separates \textbf{short affirmations} (characteristic of already-cooperative states) from \textbf{negotiation and persuasion language} (characteristic of non-cooperative states where players are trying to convince free-riders).

\paragraph{Rankings.} The top ``cooperative-direction'' messages are:
\begin{quote}
    \emph{``awesome''}, \emph{``Perfect''}, \emph{``great''}
\end{quote}
The top ``non-cooperative-direction'' messages are:
\begin{quote}
    \emph{``guys if we all put in 25 we all make more money than keeping''} \\
    \emph{``well someone is being greedy\ldots\ maybe we all lower down to 2''}
\end{quote}
The non-cooperative-direction messages are actually \emph{cooperative appeals}---players trying to convince others to contribute.

\paragraph{Probe phrase validation.} The probe phrases confirm the problem:

\begin{center}
\begin{tabular}{lc}
\toprule
Probe Phrase & Cosine Similarity \\
\midrule
``I'm going to free ride'' & $+0.04$ \\
``put in everything'' & $-0.02$ \\
``we should cooperate'' & $-0.08$ \\
``I don't trust them'' & $-0.12$ \\
``let's all contribute 25'' & $-0.13$ \\
``I'll put in zero'' & $-0.19$ \\
\bottomrule
\end{tabular}
\end{center}

``I'm going to free ride'' scores as the \emph{most} cooperative-direction probe, while ``let's all contribute 25'' scores non-cooperative. This inversion occurs because the player-level cooperative state label reflects the \emph{situation} (are others cooperating?), not the \emph{content} of the message. In cooperative states, players affirm; in non-cooperative states, they negotiate.

\paragraph{Interpretation.} The cooperative direction vector captures a real and meaningful axis---\textbf{settled cooperation versus active negotiation}---but it is not the axis we intended. This finding itself is informative: it reveals that the linguistic signature of cooperative states is \emph{absence of explicit cooperation talk}, while non-cooperative states are marked by \emph{more} cooperation-related language as players attempt to restore cooperation.

\subsection{Logistic Regression}

Despite the direction problem, embeddings still outperform VADER for predicting cooperative state:

\section{Results: Promise Direction}
\label{sec:promise}
%% ============================================================

\subsection{Direction Vector Validation}

In contrast to the cooperative direction, the promise direction captures the intended construct. The probe phrases rank in the expected order:

\begin{center}
\begin{tabular}{lcc}
\toprule
Probe Phrase & Small & Large \\
\midrule
``I promise to put in 25'' & $+0.47$ & $+0.40$ \\
``let's all commit to the max'' & $+0.25$ & $+0.29$ \\
``we all agreed to cooperate'' & $+0.22$ & $+0.25$ \\
``trust me I'll do my part'' & $+0.21$ & $+0.22$ \\
``I swear I'll put in everything'' & $+0.19$ & $+0.19$ \\
``I'll contribute the full amount'' & $+0.19$ & $+0.26$ \\
``I said I would contribute'' & $+0.15$ & $+0.18$ \\
``I'll do whatever I want'' & $+0.15$ & $+0.17$ \\
``no guarantees from me'' & $+0.13$ & $+0.11$ \\
``I might change my mind'' & $+0.11$ & $+0.10$ \\
``I never said I'd do anything'' & $+0.08$ & $+0.14$ \\
``I'm not making any promises'' & $+0.08$ & $+0.11$ \\
\bottomrule
\end{tabular}
\end{center}

The strongest probe (``I promise to put in 25'') scores +0.47, while the weakest (``I'm not making any promises'') scores +0.08. This monotonic ordering from explicit commitment language to non-commitment language confirms the direction vector captures promise-making intent.

\subsection{Rankings}

\paragraph{Top promise-direction texts (player-round level):}
\begin{quote}
    \emph{``lets do 25 yes i will''} \\
    \emph{``yeah lets do 25 all''} \\
    \emph{``can we all do 25 yes same lets do it''}
\end{quote}

\paragraph{Top no-promise-direction texts:}
\begin{quote}
    \emph{``A had to ruin it''} \\
    \emph{``like fr one selfish person got to ruin it''} \\
    \emph{``why did the balloon animal skip class he was scared of a pop quiz''}
\end{quote}

The promise direction successfully surfaces explicit commitment language at one end and off-topic or complaint-driven language at the other.

\subsection{Logistic Regression}

Embeddings dramatically outperform VADER for predicting promise-making:

\begin{table}[H]
\caption{Promise-making prediction: 5-fold CV logistic regression (667 group-rounds)}
\label{tab:promise_regression}
\input{promise_embedding_regression_comparison}
\end{table}

The 1-dimensional promise projection score alone achieves AUC = 0.90, compared to VADER's 0.67. The full embedding (PCA 50) provides marginal additional benefit (AUC = 0.89). Adding VADER to the projection score yields the highest AUC (0.90), suggesting VADER contributes minimal information beyond what the projection captures.

This is the strongest result: a single scalar projection onto the promise direction is sufficient to predict whether a group's chat contained promises with 90\% discrimination ability.

Unlike the cooperative and promise labels which reflect individual player states, homogeneity is a \textbf{group-level} property that captures the degree of contribution coordination.

\section{Results: Round-Level Liar Direction}
\label{sec:rliar}

\subsection{Direction Vector Definition}

The round-level liar direction separates players who lied about their contribution in a given round from those who did not. A player is labeled a ``round liar'' if they promised to contribute $\geq 20$ ECU but actually contributed $< 20$. Unlike the cooperative and promise labels, this label requires both a promise \emph{and} a subsequent deviation, capturing a specific form of strategic deception. With only 94 round-liar observations, the class imbalance is substantial.

\section{Comparing the Five Directions}
%% ============================================================

\begin{table}[H]
\centering
\caption{Summary comparison of direction vectors}
\label{tab:comparison}
\footnotesize
\begin{tabular}{lccccc}
\toprule
& Cooperative & Promise & Homogeneity & Round Liar & Cum.\ Liar \\
\midrule
Label level & Player & Player-round & Group-round & Player-round & Player-round \\
Label source & player\_state & promise\_class. & contributions & behavior\_class. & behavior\_class. \\
Probe validation & Inverted & Correct & --- & --- & --- \\
Positive class $n$ & 3,800 & 2,195 & --- & 94 & 123 \\
Projection-only AUC & 0.76 & 0.90 & --- & --- & --- \\
VADER-only AUC & 0.62 & 0.67 & --- & --- & --- \\
What it captures & Settled vs.\ negot. & Commitment & Coordination & Deception & Liar trait \\
\bottomrule
\end{tabular}
\end{table}

The cooperative direction's failure to capture cooperative intent---and the promise direction's success in capturing promise intent---likely stems from the nature of the labels. The cooperative state label reflects the \emph{environment} (others' behavior), which correlates with the \emph{absence} of cooperation talk. The promise label reflects the \emph{content} of the player's own speech, which the embedding naturally captures. The homogeneity direction captures a distinct group-level phenomenon: whether the group's communication reflects coordination on a common contribution level. The two liar directions test whether the embedding space can detect deceptive behavior---either at the round level (did the player lie \emph{this round}?) or cumulatively (has the player \emph{ever} lied?). The substantial class imbalance in both liar labels (94 and 123 positive cases respectively) makes these directions more challenging to validate.

\section{Implementation Details}
%% ============================================================

\subsection{Pipeline Architecture}

\begin{enumerate}
    \item \texttt{compute\_embeddings.py}: Calls OpenAI API, caches Parquet files (run once).
    \item \texttt{analyze\_embeddings.py}: Loads cached embeddings, computes cooperative centroids, direction vectors, projections, probe validation. Outputs \texttt{embedding\_projections.csv}.
    \item \texttt{analyze\_promise\_embeddings.py}: Same pipeline using promise labels. Outputs \texttt{promise\_embedding\_projections.csv}.
    \item \texttt{analyze\_homogeneity\_embeddings.py}: Same pipeline using contribution homogeneity labels. Outputs \texttt{homogeneity\_embedding\_projections.csv}.
    \item \texttt{analyze\_round\_liar\_embeddings.py}: Same pipeline using round-level liar labels (\texttt{lied\_this\_round\_20}). Outputs \texttt{round\_liar\_embedding\_projections.csv}.
    \item \texttt{analyze\_cumulative\_liar\_embeddings.py}: Same pipeline using cumulative liar labels (\texttt{is\_liar\_20}). Outputs \texttt{cumulative\_liar\_embedding\_projections.csv}.
    \item \texttt{embedding\_plots.py} / \texttt{promise\_embedding\_plots.py}: UMAP, t-SNE, projection distributions.
    \item \texttt{embedding\_regression.py} / \texttt{promise\_embedding\_regression.py} / \texttt{homogeneity\_embedding\_regression.py} / \texttt{round\_liar\_embedding\_regression.py} / \texttt{cumulative\_liar\_embedding\_regression.py}: Logistic regression model comparison.
    \item \texttt{merge\_panel\_data.py}: Integrates 20 projection score columns (4 cooperative + 4 promise + 4 homogeneity + 4 round-liar + 4 cumulative-liar) into the panel dataset.
\end{enumerate}

\section{Next Steps}
%% ============================================================

\begin{enumerate}
    \item \textbf{Alternative cooperative labels.} Try labeling by the player's own contribution level (high vs.\ low contributor) rather than others' behavior. This would make the label reflect the player's own action, which may better align with their chat content.
    \item \textbf{Panel regression.} Use the projection scores as independent variables in fixed-effects panel regressions predicting contribution.
    \item \textbf{External validation.} Project text from external datasets (Diplomacy, CaSiNo, Persuasion for Good) onto the promise direction to test whether it generalizes beyond this experiment.
    \item \textbf{Promise $\times$ cooperative interaction.} Examine how promise direction projections vary across cooperative and non-cooperative states---do promises carry different semantic content in different environments?
\end{enumerate}
